## 📋 项目概述

### 项目目标
将经典桌游《璀璨宝石 (Splendor)》电子化，作为Python课程设计项目。

**核心需求：**
- ✅ 完整实现游戏规则（2/3/4人游戏）
- ✅ 图形界面（pygame）
- ✅ 支持本地多人游戏
- ✅ 可缩放界面和全屏支持
- ✅ 规则100%符合官方规则
- ✅ 简洁现代化UI设计
- ✅ 完善的交互验证和提示系统

- ⏳ 打包成可执行文件（.exe）

- Python 3.10
- pygame 2.6.1
- openpyxl（Excel数据读取，可选）
- pyinstaller（打包）

## 🏗️ 项目架构

### 架构模式
采用**分层架构**（MVC变体）+ **可缩放UI系统** + **动态布局系统**：

```
┌─────────────────────────────────┐
│   UI Layer (界面层)              │
│   - renderer.py (渲染+缩放)     │
│   - components.py (UI组件)      │
│   - event_handler.py (交互)     │
│   - 底部提示框系统 (统一提示)    │
├─────────────────────────────────┤
│   Core Layer (核心逻辑层)        │
│   - game_engine.py (游戏引擎)   │
│   - actions.py (动作执行)       │
│   - validator.py (规则验证)     │
├─────────────────────────────────┤
│   Models Layer (数据层)          │
│   - Card, Noble, Player         │
│   - GameState                   │
├─────────────────────────────────┤
│   Utils Layer (工具层)           │
│   - data_loader.py (数据加载)   │
│   - helpers.py (辅助函数)       │
│   - scaler.py (UI缩放器)        │
└─────────────────────────────────┘
```

### 数据流
```
用户操作 → EventHandler → 验证 → GameEngine → Validator → ActionExecutor
                ↓           ↓                              ↓
          底部提示框 ←────────┘                            ↓
            Renderer ← UIScaler ← GameState ← (更新数据)
```

#### 2. **动态布局系统** ✓
**文件：** `config.py`

**实现内容：**
- ✅ **基于卡牌高度的动态计算**
  - 修改 `CARD_HEIGHT` 后，宝石和按钮区域自动下移
  - 自动计算 `CARD_AREA_HEIGHT`
  - 动态计算 `GEM_AREA_Y` 和 `BUTTON_AREA_Y`

**计算逻辑：**
```python
# ============ 卡牌尺寸配置 ============
# 修改卡牌大小时，只需修改下面两个参数，其他区域会自动调整位置
CARD_WIDTH = 120   # 卡牌宽度
CARD_HEIGHT = 175  # 卡牌高度

# 动态计算
CARD_ROW_HEIGHT = CARD_HEIGHT + 20          # 195
CARD_AREA_HEIGHT = CARD_ROW_HEIGHT * 3      # 585

CARD_AREA_Y = 20                            # 贴近顶部
GEM_AREA_Y = CARD_AREA_Y + CARD_AREA_HEIGHT + 20    # 625（自动）
BUTTON_AREA_Y = GEM_AREA_Y + GEM_AREA_HEIGHT + 10   # 735（自动）
```

**优势：**
- 后续修改卡牌大小时，无需手动调整下方区域位置
- 确保布局始终合理，不会出现重叠或过大间距

**代码位置：**
- config.py:78-121 - 动态布局计算系统

## 📝 游戏规则摘要

### 游戏目标
首先达到15分的玩家触发游戏结束，最高分者获胜。

### 玩家行动（每回合选一个）
1. **拿取宝石**
   - 拿3个不同颜色，或
   - 拿2个相同颜色（该颜色≥4个），或
   - 拿2个不同颜色（宝石种类不足时），或
   - 拿1个宝石（只剩1种时）
   - 不能拿黄金

2. **保留卡牌**
   - 保留1张卡牌（从桌面或牌堆顶）
   - 获得1个黄金
   - 最多保留3张

3. **购买卡牌**
   - 购买桌面卡或自己的保留卡
   - 支付宝石（红利可抵扣）
   - 黄金可当任意颜色
   - 获得卡牌的红利和分数

### 特殊规则
- 宝石上限：10个（超过立即弃置）
- 贵族拜访：满足条件时玩家选择一个（3分）
- 游戏结束：达到15分后所有玩家再执行最后一回合
- 平局：分数相同时，卡牌数少者获胜
