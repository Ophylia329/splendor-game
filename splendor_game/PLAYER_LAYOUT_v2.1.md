# 玩家框布局优化 v2.1

## 更新时间
2024-12-24

## 优化内容

### ✅ 1. 保留卡两列显示

**问题：**
- 原设计单列显示，最多只能显示2-3张保留卡
- 超过3张时会超出边界

**解决方案：**
- **两列布局**：第1,3,5张在第1列，第2,4,6张在第2列
- **卡牌缩小**：80×120（从90×120）
- **间距优化**：列间距8px，行间距8px
- **最多显示**：可显示6张保留卡（3行×2列）

**布局算法：**
```python
for i, card in enumerate(reserved_cards):
    col = i % 2  # 列索引：0（左列）或1（右列）
    row = i // 2  # 行索引：0, 1, 2, ...

    card_x = base_x + col * (card_width + spacing_x)
    card_y = base_y + row * (card_height + spacing_y)
```

### ✅ 2. 宝石和红利换行显示

**问题：**
- 原设计水平排列，6种宝石会占用过多宽度
- 左侧信息区域拥挤

**解决方案：**
- **每行最多3个**：宝石和红利都采用换行显示
- **自动换行**：超过3个时自动换到下一行
- **动态高度**：根据实际数量计算占用的行数

**换行算法：**
```python
gem_count_in_row = 0
for color in GEM_TYPES:
    if count > 0:
        if gem_count_in_row >= 3:
            # 换行
            gem_count_in_row = 0
            gem_x = start_x
            gem_y += row_height

        draw_gem(gem_x, gem_y)
        gem_x += spacing
        gem_count_in_row += 1

# 计算占用行数
total_rows = (total_count + 2) // 3  # 向上取整
```

---

### ✅ 3. 布局比例调整

**调整：**
- 左侧信息区：60% → **55%**
- 右侧保留卡区：40% → **45%**

**原因：**
- 保留卡改为两列后需要更多宽度
- 左侧宝石换行后可以用更窄的空间

## 技术细节

### 宝石换行计算

```python
# 显示宝石
gem_count_in_row = 0
for color in config.GEM_TYPES:
    if player.gems.get(color, 0) > 0:
        if gem_count_in_row >= 3:
            gem_count_in_row = 0
            gem_x = start_x
            gem_y += 25  # 换行

        draw_gem(gem_x, gem_y)
        gem_x += 50  # 下一个位置
        gem_count_in_row += 1

# 更新y坐标
total_gems = sum(1 for c in GEM_TYPES if gems[c] > 0)
rows = (total_gems + 2) // 3  # 向上取整
current_y += line_height + (rows - 1) * 25
```

## 空间利用

### 优化前（单列）
```
宝石区域（60%宽）：
- 6种宝石横排：需要约300px宽度
- 信息挤压，右侧空旷

保留卡区（40%宽）：
- 单列垂直：每张135高
- 3张需要405px高度（超出380px）
```

### 优化后（两列+换行）
```
宝石区域（55%宽）：
- 每行3个宝石：需要约150px宽度
- 最多2行：占用约60px高度

保留卡区（45%宽）：
- 两列并排：每行120高
- 3行可显示6张：占用约368px（不超出380px）
```
