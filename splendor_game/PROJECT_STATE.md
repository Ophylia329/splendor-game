## 📋 项目概述

### 项目目标
将经典桌游《璀璨宝石 (Splendor)》电子化，作为Python课程设计项目。

**核心需求：**
- ✅ 完整实现游戏规则（2/3/4人游戏）
- ✅ 图形界面（pygame）
- ✅ 支持本地多人游戏
- ✅ 可缩放界面和全屏支持
- ⏳ 打包成可执行文件（.exe）

**时间限制：** Python课程设计项目

**技术栈：**
- Python 3.10
- pygame 2.6.1
- openpyxl（Excel数据读取，可选）
- pyinstaller（打包）

## 🏗️ 项目架构

### 架构模式
采用**分层架构**（MVC变体）+ **可缩放UI系统**：

```
┌─────────────────────────────────┐
│   UI Layer (界面层)              │
│   - renderer.py (渲染+缩放)     │
│   - components.py (UI组件)      │
│   - event_handler.py (交互)     │
│   - scaler.py (缩放管理器)      │
├─────────────────────────────────┤
│   Core Layer (核心逻辑层)        │
│   - game_engine.py (游戏引擎)   │
│   - actions.py (动作执行)       │
│   - validator.py (规则验证)     │
├─────────────────────────────────┤
│   Models Layer (数据层)          │
│   - Card, Noble, Player         │
│   - GameState                   │
├─────────────────────────────────┤
│   Utils Layer (工具层)           │
│   - data_loader.py (数据加载)   │
│   - helpers.py (辅助函数)       │
│   - scaler.py (UI缩放器)        │
└─────────────────────────────────┘
```

### 数据流
```
用户操作 → EventHandler → GameEngine → Validator → ActionExecutor
                ↓                                       ↓
            Renderer ← UIScaler ← GameState ← (更新数据)
```

## 🔧 技术决策

### 为什么选择 pygame？
- ✅ 专为游戏设计，文档丰富
- ✅ 适合卡牌类静态游戏
- ✅ 跨平台支持
- ✅ 打包成exe简单

### 为什么实现可缩放界面？
**最新决策（2024-12-24）：**
- ✅ 适应不同屏幕分辨率
- ✅ 支持全屏和窗口模式
- ✅ 用户体验更好
- ✅ 基准分辨率1920x1080，自动缩放到任意尺寸

**实现方式：**
- UIScaler缩放管理器
- 统一缩放比例（min(scale_x, scale_y)）
- 所有UI元素动态缩放

### 为什么优先使用CSV而非Excel？
**决策（2024-12-24）：**
- ✅ 读取速度更快（无需解析ZIP格式）
- ✅ 无需openpyxl依赖
- ✅ 纯文本格式，易于调试和版本控制
- ✅ 文件更小（2KB vs 14KB）
- ✅ 跨平台兼容性更好

### 中文字体处理
- pygame默认字体不支持中文
- 使用Windows系统字体（微软雅黑/黑体/宋体）
- 实现 `get_chinese_font()` 自动检测和加载

### 窗口模式
- 默认1600x900可调整大小窗口
- F11切换全屏
- 所有UI元素自动缩放

### 自动结束回合
- 用户反馈每次都要手动点击"结束回合"很麻烦
- 执行完动作后自动结束回合
- 保留"结束回合"按钮以防万一

### 按钮闪烁修复策略
**根本原因分析：**
- 双重刷新：renderer.render() 和 main.py 都调用 flip()
- 状态更新顺序错误
- 按钮对象每帧重新创建

**解决方案：**
1. 统一刷新：只在 main.py 调用一次 flip()
2. 正确顺序：is_enabled → is_hovered → draw()
3. 对象持久化：按钮缓存复用
4. 性能优化：文字表面缓存

### UI优化设计原则
**v2.0决策（2024-12-24）：**
- 贵族多列布局节省空间
- 玩家框左右分区提高可读性
- 保留卡两列显示紧凑美观
- 宝石/红利换行节省宽度
- 黄色边框高亮更醒目

## 📝 游戏规则摘要

### 游戏目标
首先达到15分的玩家触发游戏结束，最高分者获胜。

### 玩家行动（每回合选一个）
1. **拿取宝石**
   - 拿3个不同颜色，或
   - 拿2个相同颜色（该颜色≥4个）
   - 不能拿黄金

2. **保留卡牌**
   - 保留1张卡牌（从桌面或牌堆顶）
   - 获得1个黄金
   - 最多保留3张

3. **购买卡牌**
   - 支付宝石（红利可抵扣）
   - 黄金可当任意颜色
   - 获得卡牌的红利和分数

### 特殊规则
- 宝石上限：10个（超过需弃置）
- 贵族拜访：满足条件自动获得（3分）
- 游戏结束：达到15分后所有玩家再执行最后一回合
- 平局：分数相同时，卡牌数少者获胜

## 📚 重要文件说明

### config.py
全局配置文件，包含：
- `BASE_WIDTH/HEIGHT = 1920/1080`：基准分辨率
- `DEFAULT_WINDOW_WIDTH/HEIGHT = 1600/900`：默认窗口
- `FULLSCREEN = False`：窗口模式（F11切换）
- `LAYOUT`：布局配置字典
- `CARD_WIDTH/HEIGHT = 120/175`：卡牌尺寸
- `NOBLE_SIZE = 140`：贵族尺寸
- `GEM_COLORS`：宝石颜色定义
- `MAX_GEMS_TOTAL`：根据人数配置宝石数量
- `WIN_POINTS = 15`：胜利分数
- `CARD_COUNTS`：各等级卡牌数量（40/30/20）
- `CHINESE_FONTS`：中文字体路径列表

### main.py
游戏入口，包含：
- `SplendorGame` 类：主游戏类
- `create_window()`：创建可调整大小窗口
- `toggle_fullscreen()`：F11全屏切换
- 开始菜单（按钮对象持久化）
- 主游戏循环（统一刷新管理）
- 游戏结束画面
- VIDEORESIZE事件处理

### core/game_engine.py
游戏引擎，包含：
- 游戏初始化（根据人数）
- 回合管理
- 胜利判定
- **数据加载**：
  - 优先CSV（cards.csv, nobles.csv）
  - 备用Excel（发展牌整合.xlsx, 贵族.xlsx）
  - 测试数据（24张卡，仅开发用）

### ui/event_handler.py
事件处理器，包含：
- 三种主要动作处理
- 自动结束回合逻辑
- 弃置宝石界面（按钮持久化）
- 按钮状态管理（正确的更新顺序）

### ui/renderer.py
渲染器，包含：
- 游戏画面绘制（集成UIScaler）
- `resize()`方法支持窗口大小改变
- 贵族多列布局（3个单列，4个2x2，5个3+2）
- 玩家框左右分区布局
- 保留卡两列显示
- 宝石/红利换行显示
- 黄色边框高亮当前玩家
- 布局配置（从LAYOUT读取并缩放）

### ui/components.py
UI组件，包含：
- Button：带文字缓存优化
- CardDisplay：动态间距防溢出
- GemDisplay
- NobleDisplay：增大图标和字体
- MessageBox

### utils/scaler.py
UIScaler缩放管理器，包含：
- `scale_value()`：缩放数值
- `scale_pos()`：缩放坐标
- `scale_size()`：缩放尺寸
- `scale_font_size()`：缩放字体
- `get_layout_value()`：获取缩放后的布局值
- `update_scale()`：更新缩放比例

### utils/data_loader.py
数据加载器，包含：
- `load_cards_from_csv()`
- `load_cards_from_excel()`
- `load_nobles_from_csv()`
- `load_nobles_from_excel()`
- `get_mock_cards()`：测试数据
- `get_mock_nobles()`
